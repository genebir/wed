# CLAUDE.md — 쩜오의 결혼이야기

> 예비부부 둘이 함께 보는 결혼 준비 웹서비스.
> GitHub Pages로 배포하는 정적 사이트이며, 콘텐츠(일정/체크리스트/스냅 레퍼런스)는 리포지토리 내 JSON 파일로 관리한다.

---

## 1. 핵심 정보 (하드코딩 금지 — `src/data/wedding.json`에서 관리)

| 항목 | 값 |
|---|---|
| 서비스명 | 쩜오의 결혼이야기 |
| 본식 예정일 | **2027-03-20 (토)** |
| 총 예산 | 10,000,000원 |
| 스냅 촬영 | **셀프 촬영으로 확정** (외부 업체 X) |
| 배포 | GitHub Pages (`gh-pages` 브랜치, GitHub Actions 자동 배포) |
| 사용자 | 예비 신랑/신부 2인 (비공개 성격이지만 URL 공개 사이트임을 전제) |

⚠️ 개인정보(실명, 연락처, 계좌, 상세 계약금액)는 리포에 커밋하지 않는다. 예산은 카테고리 단위 금액까지만.

---

## 2. 기술 스택

- **Vite + React 18 + TypeScript**
- **Tailwind CSS** (디자인 토큰은 `tailwind.config`에 정의)
- 상태: 콘텐츠는 정적 JSON import, 체크 상태는 `localStorage` (Phase 1)
- 라우팅: `react-router-dom` (HashRouter — GitHub Pages 새로고침 404 방지)
- 폰트: **Pretendard** (본문) + 선택적으로 세리프 디스플레이 폰트 (제목용, 예: Noto Serif KR)
- 아이콘: lucide-react
- 차트(예산): recharts

### 배포
- `vite.config.ts`에 `base: '/<repo-name>/'` 설정 필수
- `.github/workflows/deploy.yml`: main push → build → gh-pages 배포
- 이미지 최적화: 스냅 레퍼런스 이미지는 `public/gallery/`에 webp로 저장, 1200px 이하로 리사이즈

## 3. 페이지 구조

```
/               홈 (D-day, 이번 달 할 일 요약, 진행률)
/timeline       준비 일정 타임라인
/checklist      월별 체크리스트
/gallery        셀프 스냅 레퍼런스 갤러리 (무드/장소 필터)
/snap-plan      셀프 스냅 촬영 계획 (장소, 장비, 촬영 리스트)
/budget         예산 트래커
/vendors        업체/예약 관리
```

### 3.1 홈
- **D-day 카운터** (2027-03-20 기준, 큼직하게 히어로 영역)
- 전체 준비 진행률 (체크리스트 완료 비율 기반 프로그레스 바)
- "이번 달 할 일" 카드 — 현재 월에 해당하는 타임라인 항목 + 미완료 체크리스트
- 최근 저장한 스냅 레퍼런스 3~4장 미리보기

### 3.2 타임라인 (`src/data/timeline.json`)
확정된 일정. 세로 타임라인 UI, 지난 항목은 흐리게 + 체크 표시.

| 시기 | 할 일 |
|---|---|
| 2026-07 | 셀프 스냅 확정(완료), 반지 투어 및 구매 |
| 2026-08 | 혼주 + 신랑신부 헤어메이크업 예약 |
| 2026-09~10 | 셀프 스냅 촬영 |
| 2026-11 | 종이청첩장 제작, 지정 드레스샵 방문 |
| 2026-12 | 모바일 청첩장 제작, 청첩장 모임 |
| 2027-01 | 신랑 정장 대여, 혼주 의상 대여 |
| 2027-02 | 본식 드레스 셀렉, 부케 결정 |
| 2027-03 | 본식 (03-20) |

각 항목에 `tips` 필드로 부가 정보 노출 (아코디언):
- 반지: 투어는 예물거리(종로/청담) 2~3곳 비교, 각인/사이즈 리드타임 2~4주 고려
- 헤어메이크업: 본식 리허설 포함 여부, 혼주 메이크업 출장 여부 확인
- 청첩장: 본식 8~10주 전 주문, 하객 수 + 여유분 20% 산정
- 드레스 셀렉: 가봉 일정 역산, 피팅비 별도 여부
- 정장/혼주 의상: 대여는 본식 4~6주 전 가봉 기준

### 3.3 체크리스트 (`src/data/checklist.json`)
- 월별 탭 (2026-07 ~ 2027-03)
- 항목 구조: `{ id, month, title, detail?, category, done }`
- `done` 상태는 localStorage에 저장 (`checklist-state` 키)
- 카테고리: 촬영 / 예복·드레스 / 청첩장 / 예산 / 본식 / 행정
- **행정 카테고리 기본 항목 포함**: 혼인신고 준비, 신혼부부 전세/청약 관련 서류 확인, 예식장 최종 인원 통보(본식 2주 전), 축의금 계좌 정리, 신혼여행 여권 유효기간 확인
- 상단에 월별 진행률 표시

### 3.4 셀프 스냅 갤러리 (`src/data/gallery.json`)
**크롤링 방식이 아닌 수동 큐레이션 방식.** Pinterest/Instagram 크롤링은 약관 위반이며 유지보수 불가 → 대신:

- 데이터 구조:
```json
{
  "id": "g001",
  "image": "/gallery/hangang-sunset-01.webp",
  "moods": ["따뜻한", "필름감성"],
  "locations": ["한강", "야외"],
  "season": "가을",
  "timeOfDay": "골든아워",
  "sourceUrl": "https://... (원본 출처 링크)",
  "memo": "역광 + 85mm 느낌"
}
```
- 필터: 무드(멀티 선택) × 장소(멀티 선택), 칩(chip) UI
- Masonry 그리드 레이아웃, 클릭 시 라이트박스 (출처 링크 + 메모 표시)
- 무드 프리셋: 따뜻한 / 청량한 / 필름감성 / 시네마틱 / 미니멀 / 러블리
- 장소 프리셋: 한강 / 공원·수목원 / 카페 / 바다 / 골목·도심 / 스튜디오·실내
- 초기 데이터는 placeholder 6~8장으로 시딩 (실제 이미지는 사용자가 추가)
- README에 "레퍼런스 추가 방법" 가이드 섹션 작성 (이미지 저장 → webp 변환 → JSON 한 줄 추가)

### 3.5 셀프 스냅 촬영 계획 (자체 추가 기능)
셀프 스냅 확정이므로 실행 페이지가 핵심.

- **장소 후보 리스트**: 이름, 주소, 촬영 허가 필요 여부, 골든아워 방향, 방문 메모
- **장비 체크리스트**: 카메라/렌즈, 삼각대, 릴리즈 or 인터벌 촬영, 반사판, 예비 배터리·SD카드
- **샷 리스트**: 갤러리에서 참고한 레퍼런스를 연결(`galleryRef` 필드)해 "이 컷 찍자" 목록 구성
- 촬영일 체크: 일몰 시간 참고 링크, 우천 시 대체 플랜 메모 필드

### 3.6 예산 트래커 (`src/data/budget.json`)
- 총 예산 1,000만원 기준
- 카테고리: 반지 / 헤어메이크업 / 스냅(셀프—장비·소품) / 청첩장 / 정장·혼주의상 / 드레스·부케 / 본식 기타 / 예비비
- 항목별 `{ category, planned, spent, memo }`
- 도넛 차트(카테고리 비중) + 잔여 예산 프로그레스 바
- 초과 카테고리는 경고 색상 표시

### 3.7 업체/예약 관리 (`src/data/vendors.json`)
- `{ name, category, status(알아보는중/상담예약/계약완료), visitDate?, memo, link? }`
- 칸반 느낌의 상태별 컬럼 or 리스트 + 상태 뱃지

## 4. 디자인 시스템

**애플 홈페이지의 디자인 언어 차용 + "결혼 준비가 즐겁다"는 무드.**

### 컬러 토큰
```js
colors: {
  blush:  { 50:'#FDF6F5', 100:'#FAEAE8', 200:'#F5D5D2', 400:'#E8AFA9' }, // 아주 연한 핑크
  beige:  { 50:'#FAF7F2', 100:'#F3EDE3', 200:'#E8DECD', 400:'#C9B896' },
  ink:    '#1D1D1F',   // 애플 본문 텍스트 컬러
  muted:  '#6E6E73',
}
```
- 배경: `beige-50` 기본, 섹션 교차로 `white` / `blush-50`
- 포인트: `blush-400`은 버튼·프로그레스 등 액센트에만 절제해서 사용

### 애플 스타일 원칙
- **넉넉한 화이트스페이스**, 섹션 간 여백 크게 (`py-20` 이상)
- 큰 타이포 히어로: D-day는 `text-6xl~8xl`, font-weight 700, 자간 타이트(`tracking-tight`)
- 카드: `rounded-2xl` + 아주 옅은 그림자 (`shadow-sm`), 테두리 최소화
- 스크롤 진입 시 부드러운 fade-up 애니메이션 (IntersectionObserver, 과하지 않게)
- 내비게이션: 상단 고정, 반투명 블러 (`backdrop-blur`), 얇은 하단 보더
- 마이크로 인터랙션: 체크 시 살짝 튀는 스프링 애니메이션, hover scale 1.02 수준
- 이모지/일러스트 포인트로 "즐거움" 표현 (💍 📸 💌 등, 남발 금지 — 섹션 헤더 정도)

### 반응형
- 모바일 퍼스트. 둘이 폰으로 볼 확률이 높음
- 갤러리: 모바일 2열 / 데스크톱 3~4열
- 하단 탭바(모바일) or 상단 내비(데스크톱)

## 6. 코딩 컨벤션

- TypeScript strict 모드, 모든 데이터 파일에 대응하는 타입 정의 (`src/types.ts`)
- 컴포넌트는 함수형 + named export
- 날짜 처리는 `date-fns` 사용, KST 기준
- localStorage 접근은 반드시 `useLocalStorage` 훅 경유 (SSR 아님이지만 try-catch 필수)
- 커밋 메시지: `feat:`, `fix:`, `content:` (JSON 콘텐츠 수정은 content 프리픽스)
